# ZONE — تعليمات التشغيل

## 1) تجهيز قاعدة البيانات (مرة واحدة فقط)
1. افتح لوحة Supabase → مشروعك.
2. من القائمة الجانبية: **SQL Editor** → **New query**.
3. افتح ملف `schema.sql` وانسخ محتواه كاملاً وشغّله (Run).
4. تأكد أن الجداول ظهرت في **Table Editor**: `matches`, `match_sources`, `match_related`.

## 2) إنشاء حساب الأدمن (مرة واحدة)
1. في Supabase: **Authentication → Users → Add user**.
2. أدخل بريدك الإلكتروني وكلمة مرور قوية، واحفظ.
   (هذا هو الحساب الذي سيسجّل به الدخول على `admin.html`، وليس كلمة سر داخل الكود).

## 3) لوحة التحكم — admin.html
- افتحه في المتصفح وسجّل الدخول بالبريد وكلمة المرور اللي أنشأتهم بالخطوة السابقة.
- الجلسة تُحفظ تلقائيًا، ما تحتاج تسجّل دخول كل مرة إلا إذا ضغطت "تسجيل خروج".
- اختر النادي (ميلان / برشلونة) ثم أضف/عدّل/احذف المباريات.
- عند إضافة مباراة:
  - **بث مباشر** → أضف سيرفرات/قنوات متعددة (تظهر تحت المشغل).
  - **فيديو مسجل** → أضف روابط تحت تبويب "كاملة" و/أو "ملخص"، واختر مباريات أخرى لتظهر أسفل المشغل.
  - الفئة (category) هي التي تحدد الصف الذي تظهر فيه المباراة في الرئيسية تلقائيًا
    (مثال: أبرز المباريات / المباريات القادمة / ملخصات) — التطبيق يبني الصفوف تلقائيًا من هذا الحقل.
  - فعّل "إظهار في البنر العلوي" لأهم مباراة (مباشرة أو قادمة).

## 4) الميني تطبيق — index.html
- ملف واحد جاهز، يقرأ البيانات مباشرة من Supabase.
- يعمل كـ Telegram Mini App مباشرة (مضمّن `telegram-web-app.js`).

## 5) رفعه كـ Telegram Mini App
1. في تيليجرام: تحدث مع **@BotFather** → أنشئ بوت جديد (أو استخدم بوت موجود).
2. أرسل له `/newapp` واختر البوت، ثم اختر رابط الصفحة (Web App URL) الذي استضفت فيه `index.html`
   (يجب أن يكون رابط https حقيقي — استضفه على GitHub Pages / Netlify / Vercel، لا يعمل من ملف محلي).
3. بعد الرفع، البوت سيعطيك زر يفتح ZONE كميني آب.

## 6) رفعه على GitHub
رفع `index.html` و`admin.html` على GitHub **آمن تمامًا** — مفتاح Supabase الموجود بالكود هو مفتاح publishable/anon
مصمم أصلاً ليكون عامًا وظاهرًا في كود المتصفح (تمامًا كمفتاح Google Maps العام)، والأمان الحقيقي يأتي من صلاحيات
RLS في `schema.sql` لا من إخفاء المفتاح. لا حاجة لأي ملف `.env` أو سرّ إضافي.

خطوات الرفع:
1. أنشئ مستودع جديد على GitHub (مثال: `zone-app`).
2. من صفحة المستودع: **Add file → Upload files** واسحب `index.html` و`admin.html` و`schema.sql` و`README.md`.
3. فعّل **Settings → Pages** لتحصل على رابط https مباشر لـ `index.html` يُستخدم في BotFather.

إذا تبغى أفعل الرفع بنفسي، زودني برابط المستودع + Personal Access Token (Fine-grained, صلاحية Contents: Read & Write فقط)، وبسويها مباشرة.

## ملاحظة أمان
بعد تطبيق `schema.sql` الحالي، التعديل (إضافة/تعديل/حذف) ما يعمل إلا لمستخدم مسجّل دخول فعليًا عبر Supabase Auth
(الحساب اللي أنشأته بالخطوة 2). حتى لو فتح أي شخص رابط `admin.html`، ما يقدر يعدّل بدون تسجيل الدخول بذاك الحساب.
- خزّن بريد وكلمة مرور الأدمن بمكان آمن (مدير كلمات مرور).
- تقدر تضيف أكثر من مستخدم أدمن من نفس شاشة Authentication → Users.
- إذا حسّيت أن الحساب انكشف، غيّر كلمة المرور من نفس الشاشة فورًا.
